## Maker comment (post in first hour)

```
Hi makers — Tanish here. I built Tanker because I ship products solo and Claude Code, while powerful, was cold every session. Same conventions to re-explain. Same handoffs to re-wire. Same quality bar to re-enforce.

Tanker is a `.claude/` folder you drop into a project. 34 slash skills, 9 specialist agents, 3 always-on rules. The headline command, `/cto "<brief>"`, runs the full pipeline: brief → PRD → architect → plan → provision (real APIs) → parallel build → mvp review → deploy → monitor. Two human gates pre-qualified by review agents — you spend ~30 min total reviewing.

Differentiators vs other multi-agent frameworks (MetaGPT, AutoGen, CrewAI):
1. Ends in a deployed URL, not a code repo
2. Two human gates instead of fully autonomous
3. Cost ceiling (--max-cost-usd, default $10)
4. Resumable via state.json + messages.jsonl
5. Local semantic retrieval over your Obsidian + GitHub references
6. Opinionated quality rails (No AI Slop ban list)

Free + MIT. Five worked examples in the repo. Honest comparisons in the docs.

Most-useful feedback: would you trust two-gates-with-pre-qualification over fully autonomous, or would you prefer more gates / fewer?
```
